Entity Normalization Notes

Goal:
Implement deterministic entity normalization for the Ramayana RAG pipeline.

Do not rewrite translated visible text. Only enrich metadata.

Input:
data/intermediate/chapters_english.json

Output:
data/intermediate/chapters_english_normalized.json

Alias config:
configs/entity_aliases.yaml

The resolver must support these categories:
1. Primary figures
2. Sages/rishis
3. Deities/cosmic entities
4. Rakshasas
5. Vanaras
6. Places/rivers/forests
7. Epithets and alternate names

Rama, Sita, Lakshmana, Hanuman, Ravana, Bharata, Shatrughna, Dasharatha, Kausalya, Kaikeyi, Sumitra, Janaka

Matching rules:
- Case-insensitive matching for English.
- Exact substring matching for Telugu.
- Avoid false positives inside unrelated English words.
- Prefer longer aliases before shorter aliases.
- Normalize whitespace.
- Do not modify visible text.

Metadata:
At chapter level add or update:
canonical_entities, entity_mentions, places, unmapped_entity_candidates

At question level add or update:
key_entities, entity_mentions

Each entity mention:
alias, canonical, field, category, match_type
